# 设计文档目录

这个目录用于承接 `study` 的学习结论，把原理映射成当前 `Kratos` 仓库里的具体建模方案。

## 目录职责

- `study`
  - 解决：源码里发生了什么、为什么这样设计
- `design`
  - 解决：在当前 `api/service/biz/data` 结构里怎么落
- `homework`
  - 解决：下一步先写什么
- `reviews`
  - 解决：实现是否达标

## 设计文档应包含

- 当前轮目标
- Kratos 分层边界
- proto contract
- `service/biz/data` 职责划分
- 核心实体与 repo 接口
- 与 `Nacos` 上游链路的映射
